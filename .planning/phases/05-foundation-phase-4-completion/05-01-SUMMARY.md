## Decisions Made

- **Toaster outside AppShell:** `<Toaster>` placed as sibling of `<AppShell>` but both inside `<ConfirmDialogProvider>` — prevents z-index/stacking context issues with the AppShell's layout
- **useFlashToast at layout level only:** Called once in `app-header-layout.tsx`. Flash prop only changes on Inertia navigation, so a single `useEffect([flash])` at layout is sufficient; calling it in page components would risk duplicate toasts
- **Lazy closures in middleware:** Used `fn () => $request->session()->get('success')` not direct access — closures evaluated at response time (after redirect session write completes)
